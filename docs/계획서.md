# 웹 뮤직 팩토리 (Web Music Factory) — 계획서

## 1. 프로젝트 개요

Chrome Music Lab [Song Maker](https://musiclab.chromeexperiments.com/Song-Maker/)에서 영감을 받아, **음표 길이(온음표~16분음표)** 를 직접 다루며 악보를 만들고 연주하는 **교육용 정적 웹앱**이다.

수업에서 학생들이 리듬과 멜로디를 시각적으로 배치하고, 즉시 소리로 확인할 수 있도록 한다.

## 2. 목표 (현재 구현 상태)

- 피아노롤 그리드에 음을 배치하고 Tone.js로 연주
- 음표 길이: 온음표 · 2분 · 4분 · 8분 · 16분 (더블클릭 분할, 드래그 병합)
- **피아노(가락)** / **큰북(타악기)** 두 트랙
- **박자 선택**: 2/4, 3/4, 4/4, 6/8 — 변경 시 음표 dur 유지
- 데이터 모델 **v3** (박 기준 분수 `dur`)
- 빌드 없이 정적 파일만으로 실행

## 3. 핵심 인터랙션

| 동작 | 설명 |
|------|------|
| 음 배치 (피아노) | Y축 건반(C3~C5), 칸 클릭으로 on/off. 같은 박에 여러 음(화음) 가능 |
| 음 배치 (큰북) | 단일 행, 칸 클릭으로 타격 on/off |
| 칸 분할 | 더블클릭 → dur 절반 (4분→8분→16분) |
| 칸 병합 | 인접 칸 드래그 → [OK] → dur 합산 |
| 박자 변경 | 설정에서 2/4·3/4·4/4·6/8 선택, 음표 흐름 유지 |
| 마디 추가 | 오른쪽 **+** 버튼 |
| 재생/정지 | ▶ / ■, 재생 중 칸 반짝임 |
| 저장 | localStorage 자동, JSON, URL 공유 |

## 4. 기술 스택

| 항목 | 선택 |
|------|------|
| 프론트엔드 | 순수 HTML / CSS / JavaScript |
| 오디오 | Tone.js 14.x (CDN) |
| 배포 | 정적 파일 (빌드 불필요) |

## 5. 파일 구조

```
web-music-factory/
├── index.html
├── css/style.css
├── js/
│   ├── model.js       # 박자·음역·악기 상수
│   ├── note-data.js   # v3 곡 데이터 (dur, CRUD, 직렬화)
│   ├── schedule.js    # dur → 재생 이벤트(초)
│   ├── audio.js       # Tone.js 연주
│   ├── grid.js        # 피아노롤 UI
│   ├── storage.js     # 저장/불러오기
│   └── main.js        # 앱 진입점
├── scripts/
│   └── test-migration.js
├── docs/
│   ├── 계획서.md
│   ├── 데이터모델.md
│   └── 아키텍처.md
└── README.md
```

상세 아키텍처는 [아키텍처.md](아키텍처.md), 데이터 명세는 [데이터모델.md](데이터모델.md) 참고.

## 6. 악기 확장

`model.js`의 `INSTRUMENTS`에 항목을 추가하고 `createDefaultSong()`의 tracks에 ID를 넣으면 된다.

| 단계 | 악기 |
|------|------|
| 현재 | 피아노, 큰북 |
| 예정 | 소고, 트라이앵글, 실로폰 등 |

## 7. 완료된 작업

1. 피아노롤 + per-pitch lane 데이터 모델
2. 분할/병합, 마디 추가, Song Maker 스타일 UI
3. 박자 선택 (2/4, 3/4, 4/4, 6/8)
4. v3 beatFraction 데이터 모델 + v2 자동 업그레이드
5. note-data / schedule / audio 레이어 분리
6. 선형 dur 재포장 박자 마이그레이션

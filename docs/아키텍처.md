# MukeBox — 아키텍처

## 레이어 구조

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│  grid.js    │────▶│ note-data.js │◀────│ storage.js  │
│  (편집 UI)  │     │  (곡 데이터)  │     │ (저장/불러오기)│
└─────────────┘     └──────┬───────┘     └─────────────┘
                           │
                    ┌──────▼───────┐     ┌─────────────┐
                    │ schedule.js  │────▶│  audio.js   │
                    │ (시간 변환)   │     │ (Tone.js)   │
                    └──────────────┘     └─────────────┘
         model.js (상수: 박자, 음역, 악기)
```

| 파일 | 역할 |
|------|------|
| `model.js` | `TIME_SIGNATURES`, `PITCHES`, `INSTRUMENTS` 등 공유 상수 |
| `note-data.js` | v3 곡 데이터 — `dur` 분수 칸, 분할/병합, 박자 변경, JSON 직렬화 |
| `schedule.js` | `dur` + tempo + 박자 → 절대 초 재생 이벤트 |
| `audio.js` | Tone.js 합성·Transport, 재생 중 셀 하이라이트 |
| `grid.js` | 피아노롤 UI 렌더링, 클릭/분할/병합 이벤트 |
| `storage.js` | localStorage, JSON 파일, URL hash 공유 |
| `main.js` | 초기화, 박자 선택, 재생/저장 버튼 연결 |

## 스크립트 로드 순서 (`index.html`)

```
model.js → note-data.js → schedule.js → storage.js → audio.js → grid.js → main.js
```

`model.js`가 먼저 로드되어 `window.WMF`에 상수를 등록하고, `note-data.js`가 곡 API를 덮어씁니다.

## 데이터 흐름

### 편집

1. 사용자가 그리드에서 칸 클릭 → `toggleLaneCell` / `togglePercussionHit`
2. `grid.js`가 `onChange` 콜백 → `storage.js` localStorage 저장

### 박자 변경

1. `setTimeSignature(song, num, den)` 호출
2. 트랙별 칸을 시간순 선형 배열로 펼침
3. 새 `beatCount`로 마디 재포장 (`dur` 값 유지, 마디 경계에서만 split)

### 연주

1. `Schedule.buildSchedule(song)` → `{ events, cellSchedule, totalDuration }`
2. `audio.js`가 events를 Tone.Transport에 등록
3. RAF 루프로 `cellSchedule`과 Transport 시간을 비교해 하이라이트
